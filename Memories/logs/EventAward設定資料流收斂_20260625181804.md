# EventAward設定資料流收斂

## 任務範圍

- 檢查 `EventAwardController.GetAwardSettings` 是否為不必要的獨立 action。
- 將獎項頁需要的活動層級設定收斂回 `GetModel` / `GetNewModel` 回傳資料。

## 主要變更

- 移除 `EventAwardController.GetAwardSettings`。
- 在 `EventAwardViewModel` 新增 `AwardSettings`，承載活動的獎項設定欄位。
- 在 `EventAwardService.SetModel` 內補 `AwardSettings`，讓 Edit 的 `GetModel` 直接取得設定。
- Override `EventAwardController.GetNewModel()`，從 request 讀取 `pid`，讓 Create 的新 model 也能直接帶 `AwardSettings` 與 `EventId`。
- `EventAward/Content.cshtml` 移除 `GetAwardSettings` 額外呼叫，改用 `model.AwardSettings`。

## 驗證結果

- 執行 `dotnet build SmartExpoIoT/SmartExpoIoT.csproj` 成功。
- 建置結果：0 errors；warnings 為既有套件相容性、nullable、obsolete、未使用變數等警告。

## 記憶重點

- EventAward 活動層級設定不需要獨立 `GetAwardSettings` action；Edit 可由 `GetModel` 帶出，Create 可由 `GetNewModel` 帶 `pid` 後產生。
