# 前台登入客製欄位修復

時間：2026-06-25 17:31:26

## 任務範圍

- 檢查 `/EventsAdmin/Content` 的「前台登入客製設定」頁籤欄位無法顯示問題。
- 範圍限於共用 `Views/Shared/vue-component/EventContent.cshtml` 的 Vue component 渲染。

## 主要變更

- 在 `eventContentComponent` 內註冊 `v-form-file` 與 `v-form-text` 子元件，修正抽成共用元件後子元件未解析、欄位只留下自訂 tag 的問題。
- 補 `ensureCustomLogin()`，在 mounted 與 model watcher 內確保 `model.CustomLogin` 及其 `BannerPath`、`Desciption`、`Tip` 可被 `v-model` 綁定。
- 將前台登入客製 tab 的判斷集中到 `hasCustomLogin` computed。

## 驗證結果

- `dotnet build SmartExpoIoT\SmartExpoIoT.csproj` 成功，只有既有套件與 nullable 等 warning。
- 啟動 `https://localhost:5001`，Playwright 登入後進入 `/EventsAdmin/Create`，勾選「前台登入客製」並切換頁籤。
- 驗證結果：`fileInputCount = 1`、`descriptionExists = true`、`tipExists = true`，labels 為「照片上傳」「活動敘述」「提示字」，且無 consoleErrors / pageErrors。
- 驗證後已停止臨時站台並清理臨時 log。

## 記憶重點

- `EventContent.cshtml` 是獨立 Vue component；放在它 template 內的 `v-form-*` 子元件需要在該 component 自己的 `components` 註冊，不能只依賴外層 form app 的 mixin 註冊。
