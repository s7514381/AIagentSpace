## 任務範圍

## 問題根因

EventAward 資料表中 72 筆記錄的 `LotteryType` 欄位為 **NULL**。

當使用者在獎項清單頁選取「抽獎獎項」篩選時，SQL 條件為 `WHERE LotteryType = 1`，NULL 記錄全部被排除。

**前後端邏輯皆無錯誤**，純粹為既有舊資料缺少 LotteryType 值。

## 執行項目

1. **SQL 修復**：`UPDATE EventAward SET LotteryType = 1 WHERE LotteryType IS NULL AND Status != -1` → 72 筆更新 ✅
2. **程式碼路徑檢查**：

| 入口 | 是否允許 NULL | 說明 |
|------|-------------|------|
| CreateModel (表單新增) | ❌ 已阻擋 | `ModelValidation` 檢查 `LotteryType.HasValue` |
| EditModel (表單編輯) | ❌ 已阻擋 | 同上 |
| CopyModel (複製) | ❌ 已阻擋 | 同上 |
| ImportAsync (匯入) | ❌ 已阻擋 | Import 明確從 Excel 解析並設定 LotteryType |

所有 CRUD 入口都有 `ModelValidation` 保護，不會讓 NULL 寫入。
